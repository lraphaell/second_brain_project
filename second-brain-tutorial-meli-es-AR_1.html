01 Qué es y cómo funciona

El Second Brain es un sistema de conocimiento personal conectado a Claude que funciona en todas las superficies de Claude Desktop: Chat, Cowork y Code. Busca automáticamente en Slack, Confluence, Jira, Gmail y RAG, y guarda notas estructuradas en Obsidian.

📥 Captura

Búsqueda automática en Slack, Confluence, Jira, Gmail y RAG cuando preguntás algo. Cuando es valioso, pedile que lo guarde.

🧠 Razonamiento

Conecta información de distintas fuentes, identifica patrones, contradicciones y sugiere conexiones.

📝 Almacenamiento

Todo en Markdown en Obsidian — local, sin vendor lock-in, con Graph View mostrando conexiones.

🔄 Evolución

Cuanto más lo usás, más rico se pone el vault. Claude aprende de tus notas.

03 Instalación paso a paso

1
Clonar el vault template
Windows
macOS
  1. Abrí PowerShell o Git Bash
  2. Cloná el repositorio:
cd %USERPROFILE%
git clone https://github.com/lraphaell/second_brain_project.git second-brain
¿No tenés Git? Descargá el ZIP en github.com/lraphaell/second_brain_project → "Code" → "Download ZIP". Extraelo en C:\Users\TU_USUARIO\second-brain
cd ~
git clone https://github.com/lraphaell/second_brain_project.git second-brain
¿Sin Git? Descargá el ZIP y extraelo en ~/second-brain
2
Verificar la estructura

La carpeta second-brain debe estar en tu carpeta de usuario (home):

second-brain/
├── .claude/ ← CLAUDE.md (tu contexto personal)
├── 00-Inbox/ ← Captura rápida
├── 01-Projects/ ← Proyectos activos
├── 02-Areas/ ← Responsabilidades continuas
├── 03-Resources/ ← Material de referencia
├── 04-Archive/ ← Completado/inactivo
├── Daily/ · Decisions/ · Templates/
├── README.md · setup.sh
¿Por qué en la home? El vault queda "por encima" de cualquier proyecto — decisiones y aprendizajes sirven para todos. Windows: C:\Users\TU_USUARIO\second-brain · macOS: ~/second-brain
3
Personalizar el CLAUDE.md

Abrí .claude/CLAUDE.md adentro del vault y completá tus datos (nombre, rol, equipo, proyectos).

Windows: La carpeta .claude es oculta. En el Explorador: Ver → Elementos ocultos.
macOS: En el Finder: Cmd+Shift+.

05 Configurar Claude Desktop (MCP)

1
Verificar Node.js
node --version  # Necesitás v18+

Si no lo tenés: nodejs.org (versión LTS)

2
Editar la configuración de Claude Desktop
  1. Abrí Claude Desktop → Settings → Developer → Edit Config
  2. Borrá el contenido y pegá el JSON de abajo:
Windows
macOS
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": [
        "-y",
        "@bitbonsai/mcpvault",
        "C:\\Users\\TU_USUARIO\\second-brain"
      ]
    }
  }
}
⚠️ Cambiá TU_USUARIO por tu user de Windows. Usá barras dobles \\. Para saber cuál es: echo %USERPROFILE% en CMD.
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": [
        "-y",
        "@bitbonsai/mcpvault",
        "/Users/TU_USUARIO/second-brain"
      ]
    }
  }
}
⚠️ Cambiá TU_USUARIO. Usá path absoluto, no ~/. Para saberlo: echo $HOME en Terminal.
  1. Guardá el archivo
  2. Cerrá completamente Claude Desktop: Win Alt+F4 o cerrar desde la bandeja · Mac Cmd+Q
  3. Volvé a abrirlo — esperá 10-15 seg la primera vez
Si ya tenés otros MCP servers: Agregá "obsidian": {...} adentro de "mcpServers" existente, separado con coma.

10 Troubleshooting

❓ No aparece el MCP / no se ve el ícono de herramientas

¿Reiniciaste completamente? Windows: Alt+F4 o cerrar desde la bandeja del sistema. macOS: Cmd+Q (no solo cerrar la ventana). Abrí Settings → Developer y verificá el JSON.

❓ Error: "npx not found" o "node not found"

Instalá Node.js desde nodejs.org (LTS). Después de instalar, reiniciá Claude Desktop — el path se agrega automáticamente.

❓ Error: path no encontrado

Windows: Usá \\ (barras dobles): C:\\Users\\juan\\second-brain
macOS: Usá path absoluto: /Users/juan/second-brain (sin ~/)
Verificá: echo %USERPROFILE% (Win) o echo $HOME (Mac)

❓ No se ve la carpeta .claude

Windows: Explorador → Ver → Elementos ocultos
macOS: Finder → Cmd+Shift+.

❓ Timeout la primera vez

El npx baja el paquete la primera vez (~15-30 seg). Esperá y volvé a intentar. Test manual: npx @bitbonsai/mcpvault [path]

❓ Ya tengo otros MCP servers

¡No reemplaces todo! Agregá "obsidian": {...} adentro del "mcpServers" existente, con una coma.

❓ Logs de error

Windows: %APPDATA%\Claude\logs\mcp*.log
macOS: ~/Library/Logs/Claude/mcp*.log

❓ No aparece la opción "Developer" en Settings

MCP requiere Claude Pro o Max. La versión gratuita no soporta MCP servers.
