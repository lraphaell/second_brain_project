01 O que é e como funciona

O Second Brain é um sistema de conhecimento pessoal conectado ao Claude que funciona em todas as superfícies do Claude Desktop: Chat, Cowork e Code. Ele busca automaticamente em Slack, Confluence, Jira, Gmail e RAG, e salva notas estruturadas no Obsidian.

📥 Captura

Busca automática em Slack, Confluence, Jira, Gmail e RAG quando você pergunta algo. Quando é valioso, peça para salvar.

🧠 Raciocínio

Conecta informações de diferentes fontes, identifica padrões, contradições e sugere conexões.

📝 Armazenamento

Tudo em Markdown no Obsidian — local, sem vendor lock-in, com Graph View mostrando conexões.

🔄 Evolução

Quanto mais usa, mais rico o vault fica. O Claude aprende com suas notas.

03 Instalação passo a passo

1
Clonar o vault template
Windows
macOS
  1. Abra o PowerShell ou Git Bash
  2. Clone o repositório:
cd %USERPROFILE%
git clone https://github.com/lraphaell/second_brain_project.git second-brain
Sem Git? Baixe o ZIP em github.com/lraphaell/second_brain_project → "Code" → "Download ZIP". Extraia em C:\Users\SEU_USUARIO\second-brain
cd ~
git clone https://github.com/lraphaell/second_brain_project.git second-brain
Sem Git? Baixe o ZIP e extraia em ~/second-brain
2
Verificar a estrutura

A pasta second-brain deve estar na sua pasta de usuário (home):

second-brain/
├── .claude/ ← CLAUDE.md (seu contexto pessoal)
├── 00-Inbox/ ← Captura rápida
├── 01-Projects/ ← Projetos ativos
├── 02-Areas/ ← Responsabilidades contínuas
├── 03-Resources/ ← Material de referência
├── 04-Archive/ ← Concluído/inativo
├── Daily/ · Decisions/ · Templates/
├── README.md · setup.sh
Por que na home? O vault fica "acima" de qualquer projeto — decisões e aprendizados servem a todos. Windows: C:\Users\SEU_USUARIO\second-brain · macOS: ~/second-brain
3
Personalizar o CLAUDE.md

Abra .claude/CLAUDE.md e preencha seus dados (nome, cargo, time, projetos).

Windows: A pasta .claude é oculta. No Explorador: Exibir → Itens ocultos.
macOS: No Finder: Cmd+Shift+.

05 Configurar Claude Desktop (MCP)

1
Verificar Node.js
node --version  # Precisa ser v18+

Se não tem: nodejs.org (versão LTS)

2
Editar configuração do Claude Desktop
  1. Abra o Claude Desktop → Settings → Developer → Edit Config
  2. Apague o conteúdo e cole o JSON abaixo:
Windows
macOS
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": [
        "-y",
        "@bitbonsai/mcpvault",
        "C:\\Users\\SEU_USUARIO\\second-brain"
      ]
    }
  }
}
⚠️ Troque SEU_USUARIO pelo seu user do Windows. Use barras duplas \\. Para descobrir: echo %USERPROFILE% no CMD.
{
  "mcpServers": {
    "obsidian": {
      "command": "npx",
      "args": [
        "-y",
        "@bitbonsai/mcpvault",
        "/Users/SEU_USUARIO/second-brain"
      ]
    }
  }
}
⚠️ Troque SEU_USUARIO. Use path absoluto, não ~/. Para descobrir: echo $HOME no Terminal.
  1. Salve o arquivo
  2. Feche completamente o Claude Desktop: Win Alt+F4 ou fechar na bandeja · Mac Cmd+Q
  3. Reabra — espere 10-15 seg na primeira vez
Se já tem outros MCP servers: Adicione "obsidian": {...} dentro do "mcpServers" existente, separado por vírgula.

10 Troubleshooting

❓ MCP não aparece / ícone de ferramentas não aparece

Reiniciou completamente? Windows: Alt+F4 ou fechar na bandeja. macOS: Cmd+Q (não só fechar janela). Abra Settings → Developer e confira o JSON.

❓ Erro: "npx not found" ou "node not found"

Instale Node.js de nodejs.org (LTS). Após instalar, reinicie o Claude Desktop — o path é adicionado automaticamente.

❓ Erro: path não encontrado

Windows: Use \\ (barras duplas): C:\\Users\\joao\\second-brain
macOS: Use path absoluto: /Users/joao/second-brain (sem ~/)
Confira: echo %USERPROFILE% (Win) ou echo $HOME (Mac)

❓ Pasta .claude não aparece

Windows: Explorador → Exibir → Itens ocultos
macOS: Finder → Cmd+Shift+.

❓ Timeout na primeira vez

O npx baixa o pacote na primeira vez (~15-30s). Espere e tente de novo. Teste manual: npx @bitbonsai/mcpvault [path]

❓ Já tenho outros MCP servers

Não substitua! Adicione "obsidian": {...} dentro do "mcpServers" existente, com vírgula.

❓ Logs de erro

Windows: %APPDATA%\Claude\logs\mcp*.log
macOS: ~/Library/Logs/Claude/mcp*.log

❓ Sem opção "Developer" no Settings

MCP requer Claude Pro ou Max. A versão gratuita não suporta MCP servers.
